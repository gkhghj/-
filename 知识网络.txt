│   │   └── 方案4：伪元素清除法

│   │       │   ```css
│   │       │   .clearfix::after {
│   │       │   content: "";
│   │       │   display: block;
│   │       │   clear: both;
│   │       │   
│   │       └── 优点：无冗余DOM / 可复用 / 语义化好
│   └── 清除浮动选择策略
│       ├── 简单布局：overflow: hidden（快速但需注意溢出）

│   │   ├── static（默认值）
│   │   │   ├── 元素遵循正常文档流
│   │   │   └── 忽略top、right、bottom、left和z-index属性

│   │   │   ├── 相对于元素自身原始位置进行偏移
│   │   │   ├── 原始位置仍被占据（不脱离文档流）

│   │   │   ├── 相对于最近的非static祖先元素进行定位
│   │   │   ├── 脱离正常文档流（不占据原始空间）
│   │   │   └── 若无符合条件的祖先，则相对于<html>定位
│   │   ├── fixed（固定定位）
│   │   │   ├── 相对于浏览器窗口（视口）进行定位
│   │   │   ├── 脱离文档流，滚动页面时位置固定不变

│   │       ├── 滚动到特定位置时固定，超出后恢复相对定位

│   │   │   ├── 取值为整数，数值越大元素越靠近用户（层叠越靠前）

Flexbox 弹性布局详解
├── 1. Flexbox 基础概念
│   ├── 弹性布局核心思想
│   │   ├── 一维布局方案（专注于主轴方向的灵活排列）
│   │   ├── 解决传统布局痛点（浮动塌陷、定位复杂、百分比计算繁琐）
│   │   └── 优势：简单、灵活、响应式友好
│   ├── 弹性容器与弹性项目
│   │   ├── 弹性容器（Flex Container）：设置 display: flex 的父元素
│   │   └── 弹性项目（Flex Item）：弹性容器的直接子元素
│   └── 主轴与交叉轴
│       ├── 主轴（Main Axis）：flex-direction 定义的方向（默认水平向右）
│       └── 交叉轴（Cross Axis）：与主轴垂直的方向（默认垂直向下）
├── 2. Flex 容器属性（控制整体布局）
│   ├── flex - direction（主轴方向）
│   │   ├── 取值：row（默认，水平从左到右）、row - reverse（水平从右到左）、column（垂直从上到下）、column - reverse（垂直从下到上）
│   │   └── 作用：决定弹性项目的排列方向
│   ├── justify - content（主轴对齐方式）
│   │   ├── 取值：flex - start（默认，起始端对齐）、flex - end（末尾端对齐）、center（居中对齐）、space - between（两端对齐，项目间隔相等）、space - around（环绕对齐，项目两侧间隔相等）、space - evenly（完全均匀分布）
│   │   └── 作用：控制弹性项目在主轴方向上的对齐和分布
│   ├── align - items（交叉轴对齐方式）
│   │   ├── 取值：stretch（默认，拉伸填满交叉轴）、flex - start（交叉轴起始端对齐）、flex - end（交叉轴末尾端对齐）、center（交叉轴居中对齐）、baseline（基线对齐）
│   │   └── 作用：控制弹性项目在交叉轴方向上的对齐方式
│   ├── flex - wrap（换行规则）
│   │   ├── 取值：nowrap（默认，不换行）、wrap（换行，第一行在上方）、wrap - reverse（换行，第一行在下方）
│   │   └── 作用：决定弹性项目是否换行以及换行方向
│   ├── align - content（多行交叉轴对齐）
│   │   ├── 取值：flex - start（多行起始端对齐）、flex - end（多行末尾端对齐）、center（多行居中对齐）、space - between（多行两端对齐）、space - around（多行环绕对齐）、stretch（默认，多行拉伸填满交叉轴）
│   │   └── 作用：当有多行弹性项目时，控制多行在交叉轴方向上的对齐方式
│   ├── flex - flow（复合属性）
│   │   ├── 语法：flex - flow: <flex - direction> <flex - wrap>
│   │   └── 作用：同时设置主轴方向和换行规则
│   └── gap（间距控制）
│       ├── 取值：gap: <row - gap> <column - gap>（行间距 列间距）
│       └── 作用：设置弹性项目之间的间距（兼容性需注意）
├── 3. Flex 项目属性（控制单个项目）
│   ├── flex - grow（放大比例）
│   │   ├── 取值：非负数字（默认 0）
│   │   └── 作用：当弹性容器有剩余空间时，项目按比例放大占据空间
│   ├── flex - shrink（缩小比例）
│   │   ├── 取值：非负数字（默认 1）
│   │   └── 作用：当弹性容器空间不足时，项目按比例缩小
│   ├── flex - basis（基础尺寸）
│   │   ├── 取值：长度值（如 200px）、百分比、auto（默认，由内容决定）
│   │   └── 作用：定义项目在分配多余空间之前的默认尺寸
│   ├── flex（复合属性）
│   │   ├── 语法：flex: <flex - grow> <flex - shrink> <flex - basis>
│   │   └── 作用：同时设置放大比例、缩小比例和基础尺寸
│   ├── align - self（单个项目交叉轴对齐）
│   │   ├── 取值：auto（默认，继承父容器 align - items）、flex - start、flex - end、center、baseline、stretch
│   │   └── 作用：单独控制某个弹性项目在交叉轴方向上的对齐方式
│   ├── order（排序）
│   │   ├── 取值：整数（默认 0）
│   │   └── 作用：控制弹性项目的排列顺序（数值越小越靠前）
│   └── min - width / min - height（最小尺寸限制）
│       └── 作用：防止项目被过度缩小
├── 4. Flexbox 布局实战案例
│   ├── 导航栏布局
│   │   ├── 水平导航栏（flex - direction: row，justify - content: space - between）
│   │   ├── 垂直导航栏（flex - direction: column）
│   │   └── 响应式导航栏（结合媒体查询和 flex - wrap）
│   ├── 卡片组布局
│   │   ├── 等高卡片组（align - items: stretch）
│   │   ├── 居中对齐卡片组（justify - content: center，align - items: center）
│   │   └── 多行卡片组（flex - wrap: wrap，align - content: space - around）
│   ├── 圣杯布局（经典三栏布局）
│   │   ├── 左右固定宽，中间自适应（结合 flex - grow 和 flex - basis）
│   │   └── 使用 Flexbox 简化传统圣杯布局实现
│   ├── 表单布局
│   │   ├── 标签和输入框对齐（align - items: center）
│   │   └── 多列表单布局（flex - direction: row，flex - wrap: wrap）
│   └── 响应式网格布局
│       ├── 自适应列数网格（flex - wrap: wrap，flex - basis: 计算值）
│       └── 结合媒体查询实现不同屏幕尺寸的布局调整

CSS3 高级特性与响应式布局基础
├── 1. CSS3 动画与过渡
│   ├── 过渡（Transitions）
│   │   ├── transition - property：指定过渡效果的 CSS 属性（如 width、color 等）
│   │   ├── transition - duration：设置过渡效果的持续时间（如 0.3s、2s 等）
│   │   ├── transition - timing - function：定义过渡效果的速度曲线（如 ease、linear、ease - in - out 等）
│   │   ├── transition - delay：设置过渡效果的延迟时间（如 0.5s）
│   │   └── 作用：实现元素属性变化的平滑过渡效果（如鼠标悬停时元素的颜色、大小变化）
│   ├── 动画（Animations）
│   │   ├── @keyframes：定义动画的关键帧（指定动画在不同时间点的状态）
│   │   ├── animation - name：引用 @keyframes 定义的动画名称
│   │   ├── animation - duration：设置动画的持续时间
│   │   ├── animation - timing - function：定义动画的速度曲线
│   │   ├── animation - delay：设置动画的延迟时间
│   │   ├── animation - iteration - count：设置动画的播放次数（如 1、infinite 无限循环）
│   │   ├── animation - direction：设置动画的播放方向（如 normal、reverse、alternate 等）
│   │   ├── animation - fill - mode：设置动画结束后的状态（如 forwards、backwards 等）
│   │   └── animation：复合属性（可同时设置多个动画属性）
│   └── 动画与过渡的应用场景
│       ├── 鼠标悬停效果（如按钮颜色变化、图片缩放）
│       ├── 页面加载动画（如元素淡入淡出、滑动进入）
│       └── 交互反馈（如点击按钮后的动画提示）
├── 2. CSS3 变换（Transforms）
│   ├── 2D 变换
│   │   ├── translate（平移）：translateX()、translateY() 或 translate(x, y)（移动元素位置）
│   │   ├── scale（缩放）：scaleX()、scaleY() 或 scale(x, y)（改变元素大小）
│   │   ├── rotate（旋转）：rotate()（以元素中心为原点旋转一定角度）
│   │   ├── skew（倾斜）：skewX()、skewY() 或 skew(x, y)（使元素倾斜变形）
│   │   └── transform - origin：设置变换的原点（默认是元素中心）
│   ├── 3D 变换（基础概念）
│   │   ├── perspective（透视）：为 3D 变换提供透视效果（模拟人眼观察 3D 物体的效果）
│   │   ├── translate3d()、scale3d()、rotate3d()：3D 空间中的平移、缩放和旋转
│   │   └── 3D 变换的应用场景（如 3D 卡片翻转、3D 导航菜单）
│   └── 变换的组合与嵌套
│       ├── 多个变换函数可组合使用（如同时平移和旋转元素）
│       └── 变换可应用于嵌套元素，实现复杂的动画效果
├── 3. 媒体查询（Media Queries）
│   ├── 媒体查询原理
│   │   ├── 根据设备的特性（如屏幕宽度、高度、分辨率等）应用不同的 CSS 样式
│   │   └── 实现响应式设计，让页面适配不同设备
│   ├── 媒体查询语法
│   │   ├── @media 媒体类型 and (媒体特性) { CSS 样式 }
│   │   ├── 常见媒体类型：all（所有设备）、screen（屏幕设备）、print（打印设备）
│   │   ├── 常见媒体特性：width（屏幕宽度）、min - width（最小屏幕宽度）、max - width（最大屏幕宽度）、height（屏幕高度）、orientation（屏幕方向，portrait 竖屏、landscape 横屏）
│   │   └── 逻辑运算符：and（与）、not（非）、only（仅）
│   ├── 媒体查询的应用
│   │   ├── 不同屏幕尺寸的布局调整（如手机、平板、电脑的页面布局）
│   │   ├── 图片和文字的自适应缩放
│   │   └── 隐藏或显示特定设备上的元素
│   └── 响应式设计原则
│       ├── 移动优先（先设计移动端页面，再逐步适配更大屏幕）
│       ├── 弹性布局（使用弹性盒子或网格布局，让元素自适应排列）
│       └── 相对单位（使用 rem、vw、vh 等相对单位，实现页面的自适应缩放）
├── 4. 相对单位与自适应布局
│   ├── 相对单位
│   │   ├── rem：相对于根元素（<html>）的字体大小（通常设置根元素字体大小为 16px 或 10px，方便计算）

│   │   ├── vh（Viewport Height）：相对于视口高度的百分比（1vh = 视口高度的 1%）
│   │   ├── %（百分比）：相对于父元素的相应属性（如宽度、高度、字体大小等）
│   │   └── em：相对于当前元素的字体大小（常用于字体大小的相对设置）
│   ├── 自适应布局策略
│   │   ├── 使用相对单位设置元素的尺寸（如宽度、字体大小）
│   │   ├── 结合媒体查询，根据不同屏幕尺寸调整相对单位的基准值
│   │   └── 利用弹性盒子或网格布局，让元素在不同屏幕尺寸下自适应排列
│   └── 实际应用案例
│       ├── 响应式导航栏（使用相对单位和媒体查询调整导航栏的布局和样式）
│       ├── 自适应图片画廊（图片大小根据屏幕尺寸自适应缩放）

│   │   ├── meta viewport 标签
│   │   │   ├── 作用：控制视口的宽度、缩放比例等
│   │   │   ├── 常用属性：width（视口宽度）、initial - scale（初始缩放比例）、minimum - scale（最小缩放比例）、maximum - scale（最大缩放比例）、user - scalable（是否允许用户缩放）
│   │   │   └── 示例：meta name="viewport" content="width=device - width, initial - scale=1.0"
│   │   └── 视口与页面布局的关系：影响页面元素在不同设备上的显示尺寸
│   ├── 像素比（Device Pixel Ratio，DPR）
│   │   ├── 像素比概念：设备物理像素与CSS像素的比值
│   │   ├── 高像素比设备（如Retina屏）：物理像素多，显示更清晰
│   │   ├── 像素比的影响：图片模糊、边框显示不清晰等问题
│   │   └── 解决方案：使用高分辨率图片、通过CSS媒体查询适配不同像素比
│   └── 移动端特性总结
│       ├── 屏幕尺寸多样：从小型手机到大屏平板
│       ├── 操作方式不同：以触摸操作为主
│       └── 网络环境复杂：可能面临弱网、高延迟等问题
